# はじめに

## CAMERA_FRAMES とは

CAMERA_FRAMES は、3D シーンから**構図を確定させて紙面用の出力を作る**ことに特化した DCC ツールです。汎用の 3D ビューアではなく、次の 5 要素を一貫して扱うことを主眼に置いています。

- **ショットカメラ** — 構図ごとに独立したカメラオブジェクト
- **用紙** — 紙面サイズとアンカー（基準点）
- **フレーム** — 紙面上に配置する矩形（単独でも複数でも）
- **下絵** — 下絵画像（プリセット単位で管理）
- **書き出し** — PNG / PSD 書き出し

> 💡 **ビューポート**（作業用ビュー）と **カメラビュー**（構図決定ビュー）は別物として扱います。同じ画面ですが役割が違うので、章 [画面構成](02-ui-layout.md) と [ビューポートとツール](08-viewport-tools.md) を後で確認してください。

## 起動する

開発モードでは `npm run dev` で Vite サーバを立てます。ブラウザで `http://localhost:3000` を開くと CAMERA_FRAMES が起動します。

![起動直後の画面](../assets/screenshots/ja/drop-hint.png)

初回はシーンが読み込まれていないので、ビューポート上に操作ヒントが表示されます。

## 最初の 5 分

次の流れで、シーンを開いて PNG を出力するまでが試せます。

### 1. シーンを開く

操作方法は 3 通り:

- ビューポートにファイルをドロップする（`.ply` / `.spz` / `.splat` / `.kSplat` / `.sog` / `.zip` / `.rad` / `.glb` / `.gltf` に対応）
- ツールレールの `Open Files...`（`Ctrl+O`）でファイルを選ぶ
- ツールレールの URL 入力欄に HTTP(S) URL を貼って `Load` する

対応形式の詳細は [ファイルを開く・保存する](03-open-save.md) を参照してください。

![シーンを開いた直後の画面](../assets/screenshots/ja/first-scene-loaded.png)

### 2. ビューポートで構図を探す

マウスで次の操作ができます。

- 左ドラッグ: オービット（注視点を中心に回転）
- `Ctrl+` 左ドラッグ または 右ドラッグ: ヒット点を中心にアンカーオービット
- マウスホイール: ドリー（前後移動）
- 右ボタンドラッグ または `Shift+` 左ドラッグ: パン

詳しくは [ビューポートとツール](08-viewport-tools.md) を参照。

### 3. ショットカメラを作る

構図が決まったら、インスペクターのカメラタブにある **ショットカメラ** セクションでショットカメラを追加します。これで現在の視点が保存され、後から呼び戻せます。

ショットカメラは単なる視点のメモではなく、独自の紙面サイズ（用紙）を持った **構図ごとのカメラオブジェクト**です。複数のショットカメラを作って、構図のバリエーションを管理できます。

詳しくは [ショットカメラ](05-shot-camera.md) を参照。

### 4. 用紙を調整する

インスペクターの **用紙** セクションで、出力したい紙面サイズ（幅 × 高さ）とアンカー（紙面のどこを基準にするか）を調整します。

CAMERA_FRAMES では、アンカーと中心点を維持したまま紙面サイズだけを変えられます。これにより、構図を壊さずに出力領域の形状だけを変更できます。

詳しくは [用紙とフレーム](06-output-frame-and-frames.md) を参照。

### 5. 書き出しする

インスペクターの **書き出し** タブで書き出しを実行します。

- **書き出し対象** — `current`（現在のショットカメラのみ）/ `all`（全ショットカメラ）/ `selected`（選択したショットカメラのみ）
- **形式** — `png` または `psd`

`書き出し` ボタンを押すと、ショットカメラごとに用紙で定義された紙面サイズで画像が書き出されます。

詳しくは [書き出し](10-export.md) を参照。

## 保存する

CAMERA_FRAMES には 2 種類の保存があります。性質が異なるので使い分けてください。

| ショートカット | 保存先 | 用途 |
|---|---|---|
| `Ctrl+S` | **作業保存** — ブラウザの IndexedDB | 同じブラウザで作業を再開する |
| `Ctrl+Shift+S` | **パッケージ保存** — `.ssproj` ファイルとしてダウンロード | 別環境へ持ち運ぶ / バージョン管理する |

ビューポート右上の HUD に保存状態が表示されます:

- `*` — 作業保存が未保存
- `PKG` — `.ssproj` に未反映の変更あり

詳しくは [ファイルを開く・保存する](03-open-save.md) を参照。
