# 用語集とトラブルシューティング

## 1. 用語集

各用語は関連の強いカテゴリで並べています。

### プロジェクトと保存

- **プロジェクト** — ワークスペース、ショットカメラ、シーンアセット、下絵 をまとめた単位。`.ssproj` ファイルと 作業保存 で永続化される。[→ 3](03-open-save.md)
- **`.ssproj`** — CAMERA_FRAMES のプロジェクトパッケージ（ZIP 形式）。[→ 3](03-open-save.md)
- **作業保存** — ブラウザ内に保存される作業状態。`Ctrl+S`。同じブラウザでの作業再開用。[→ 3](03-open-save.md)
- **パッケージ保存** — `.ssproj` ファイルをダウンロードして保存。`Ctrl+Shift+S`。別環境へ持ち運び / バックアップ用。[→ 3](03-open-save.md)
- **旧 `.ssproj`** — 以前のバージョンで保存された `.ssproj`。そのまま開けば自動で現行フォーマットに変換されてロードされる。[→ 3](03-open-save.md)

### カメラ

- **ショットカメラ** — 構図ごとに保存するカメラオブジェクト。pose / lens / clipping / 用紙 / フレーム / 下絵 の紐付け / 書き出し設定を持つ。常に perspective。[→ 5](05-shot-camera.md)
- **ビューポート カメラ** — エディタ用の作業カメラ。ショットカメラ とは別管轄。orthographic に切替可能。[→ 5](05-shot-camera.md)
- **カメラモード / ビューポートモード** — ビューポート の描画モード。前者はアクティブな ショットカメラ の視点、後者は自由視点。[→ 2](02-ui-layout.md), [→ 5](05-shot-camera.md)
- **Equivalent MM** — 35mm 換算の焦点距離。ショットカメラ のレンズ設定で使われる。14〜200 mm。[→ 5](05-shot-camera.md)
- **ロールロック** — orbit 時に roll（前方軸回転）を固定する設定。水平線を傾けたくない時に ON。[→ 5](05-shot-camera.md)
- **書き出し名** — ショットカメラ ごとの出力ファイル名テンプレート。`%cam` 変数で ショットカメラ 名に展開される。[→ 5](05-shot-camera.md)

### 紙面と構図

- **用紙** — 紙面サイズ（width × height）と アンカー を持つ、書き出し の投影契約そのもの。カメラモード で 用紙枠 として表示される。[→ 6](06-output-frame-and-frames.md)
- **紙面サイズ** — 用紙 の幅と高さ。基準 1754 × 1240 px に対する比率。100%〜912%（幅）/ 1290%（高さ）。[→ 6](06-output-frame-and-frames.md)
- **アンカー** — 紙面の基準点。3 × 3 グリッドで 9 点のうちから選ぶ。paper resize 時の固定点として機能。[→ 6](06-output-frame-and-frames.md)
- **Custom frustum** — アンカー と中心を固定したまま 紙面サイズ だけ変えられる CAMERA_FRAMES の projection 仕組み。[→ 6](06-output-frame-and-frames.md)
- **用紙枠** — カメラモード で ビューポート 上に表示される紙面枠。8 リサイズハンドル + 4 パンエッジ + アンカー点 + メタラベル。[→ 6](06-output-frame-and-frames.md)
- **フレーム** — アニメ撮影で言う**撮影フレーム**。紙面内のどこを撮るかを矩形で示す。1 ショットカメラ につき最大 20 個。**1 つなら静止カット**、**複数ならパン・TU/TB・ズームなどのカメラワーク指示**として使う。[→ 6](06-output-frame-and-frames.md)
- **フレームマスク** — フレーム の外側を暗くする機能。モード (off/all/selected) × 形状 (bounds/軌道) × 不透明度 の組合せ。[→ 6](06-output-frame-and-frames.md)
- **軌道** — 複数 フレーム の中心を繋ぐ経路。line / spline。マスク形状や 書き出し の軌道線に使われる。[→ 6](06-output-frame-and-frames.md)
- **Spline ノードモード** — spline 軌道 時の各 フレーム のノードモード。auto / corner / mirrored / free。[→ 6](06-output-frame-and-frames.md)
- **軌道編集トグル** — 実行時のみの編集状態。ON で ビューポート 上に 軌道ハンドル が表示される。[→ 6](06-output-frame-and-frames.md)

### シーンアセット

- **シーンアセット** — 3D シーンに配置する個別オブジェクト。`スプラット` か `model` の 2 種類。[→ 4](04-scene-assets.md)
- **スプラット** — Gaussian splatting アセット。`.ply` / `.spz` / `.splat` / `.kSplat` / `.sog` / `.rad` / `.zip`。スプラット編集 対応。[→ 4](04-scene-assets.md)
- **モデル** — glTF / glb モデル。スプラット編集 非対応。[→ 4](04-scene-assets.md)
- **baseScale / worldScale** — アセット のスケールは `baseScale × worldScale` で計算される。読み込み時のスケール（baseScale）を保ったまま倍率（worldScale）だけ変更する設計。[→ 4](04-scene-assets.md)
- **Content Transform** — アセット の ラッパー と コンテンツオブジェクト の 2 段階構造の内、コンテンツオブジェクト 側の変形。通常は `Apply Transform` で確定される。[→ 4](04-scene-assets.md)
- **Working pivot** — 変形ツールの pivot（回転 / スケールの中心）。ピボットツールで編集。[→ 4](04-scene-assets.md)
- **書き出しロール** — アセット の 書き出し時の役割。`beauty`（通常）/ `omit`（別チャネル）。[→ 4](04-scene-assets.md)
- **マスクグループ** — アセット を論理的にグループ化する任意文字列。[→ 4](04-scene-assets.md)

### スプラット編集

- **スプラット編集** — スプラット 内の個別 スプラット を選択・削除・分離・複製・変形するモード。`Shift+E`。[→ 9](09-per-splat-edit.md)
- **Raw selection** — 編集セッション中のみ存在する スプラット 選択状態。実行時のみ保持され、保存されない。[→ 9](09-per-splat-edit.md)
- **Separate** — 選択 スプラット を別アセットとして切り出す操作。新アセットは元の直上に挿入される。[→ 9](09-per-splat-edit.md)
- **Box / Brush / Transform** — スプラット編集 の 3 つのツール。[→ 9](09-per-splat-edit.md)

### リファレンス画像

- **下絵** — カメラビュー 上に重ねる下絵。プリセット × アイテム × ショットカメラ 紐付け の 3 層構造。[→ 7](07-reference-images.md)
- **プリセット** — 構図ごとに使い分ける下絵セット（複数 アイテム のコンテナ）。[→ 7](07-reference-images.md)
- **`(blank)` プリセット** — 下絵セットを使っていない状態を表す空 プリセット。削除不可。[→ 7](07-reference-images.md)
- **紙面基準サイズ** — 各 プリセット が持つ、下絵を配置した時点の基準紙面サイズ。用紙のサイズが後から変わっても、下絵はこの基準で相対位置を保って再配置される。[→ 7](07-reference-images.md)
- **ショットカメラ ごとの下絵差分** — 同じ プリセット を複数 ショットカメラ で共有しつつ、ショットカメラ ごとに アイテム の位置や表示を独立して調整できる仕組み。[→ 7](07-reference-images.md)
- **Front / Back グループ** — 下絵アイテム の前後位置。シーンの前 / 後ろに描画される。[→ 7](07-reference-images.md)
- **プレビュー表示 / 書き出し参加** — アイテム ごとに独立した 2 つの切替。プレビュー と 書き出し の参加条件に関わる。[→ 7](07-reference-images.md)

### ビューポート とツール

- **パイメニュー** — 中ボタンドラッグで展開する 10 アクションのラジアルメニュー。[→ 8](08-viewport-tools.md)
- **ツールレール** — 画面左のツールパネル。ドラッグ可能。[→ 2](02-ui-layout.md), [→ 8](08-viewport-tools.md)
- **軸 Gizmo** — orthographic 時に画面隅に出る軸方向表示ウィジェット。[→ 8](08-viewport-tools.md)
- **変形 Gizmo** — 変形 / ピボットツール使用中に表示される、移動 / 回転 / スケール操作用のハンドル群。[→ 8](08-viewport-tools.md)
- **アンカー orbit** — カーソルが指すヒット点を中心に orbit する操作。`Ctrl +` 左ドラッグ または 右ドラッグ。[→ 8](08-viewport-tools.md)
- **レンズ調整 / ロール調整** — パイメニュー 経由で入るレンズ / roll のリアルタイム調整モード。Escape で抜ける。[→ 5](05-shot-camera.md)

### 書き出し

- **書き出し対象** — 出力対象の範囲。`current` / `all` / `selected`。[→ 10](10-export.md)
- **書き出し形式** — `png` または `psd`。[→ 10](10-export.md)
- **グリッドレイヤーモード** — PSD 出力時のグリッドレイヤーの位置。`bottom`（Render より下） / `overlay`（Render より上）。[→ 10](10-export.md)
- **モデルレイヤー / スプラットレイヤー** — PSD 書き出し で モデル / スプラット を個別レイヤーに分離するオプション。[→ 10](10-export.md)
- **下絵 を含める** — 書き出し 実行単位で 下絵 を含めるかの切替。[→ 10](10-export.md)
- **軌道 書き出し起点** — PSD 軌道レイヤー の軌道起点。`none` / `center` / 4 隅。[→ 10](10-export.md), [→ 6](06-output-frame-and-frames.md)
- **LoD 収束ウォームアップ** — スプラット アセットがある場合、書き出し前に最低 2 パス のウォームアップが入る。[→ 10](10-export.md)

## 2. トラブルシューティング

### 2.1 ファイルを開く / 読み込み

#### Q. ドロップしても何も起きない

- ファイル拡張子を確認してください。対応形式は [3.1.2](03-open-save.md#12-対応ファイル形式) を参照
- 大きな `.zip` / `.ssproj` の場合、展開に時間がかかっている可能性があります（進捗オーバーレイ が出るはず）

#### Q. リモート URL からロードできない

- プロトコルが `http://` または `https://` であること
- CORS の制約で URL にアクセスできない場合があります（サーバー側で Access-Control-Allow-Origin が必要）

#### Q. `?load=` の URL が自動で読み込まれない

- HTTPS 要件やプライベートホスト などの安全性検証に引っかかっている可能性があります。警告ダイアログを確認してください
- URL が許可された場合も、**Continue Load** を押して確認するまで読み込みは走りません

#### Q. 旧 `.ssproj` を開くとエラーが出る / 一部が欠ける

- 現行フォーマットで開けない場合は、自動的に旧フォーマットでの読み込みが試行されます
- 旧フォーマットから読み取れる情報は `.ssproj` 内のデータ構造に依存します。壊れた / 切り詰められた `.ssproj` は読めない場合があります

#### Q. PSD の読み込みで レイヤー が一部出てこない

- 子を持つフォルダレイヤーは抽出されません（leaf レイヤー のみ個別 アイテム として展開されます）

### 2.2 保存 / 復元

#### Q. `.ssproj` を開いたが、前の編集内容が復元されない

作業保存 は元になった `.ssproj` と紐付けて保持されます。別の `.ssproj` を開いた場合や、`.ssproj` を別バージョンで上書きしていた場合は、対応する 作業保存 が見つからず復元されません。

#### Q. `Ctrl+S` を押したのにファイルがダウンロードされた

新規プロジェクト（まだ一度も `.ssproj` を保存していない状態）では、`Ctrl+S` が **パッケージ保存 にフォールバック**します。意図せず パッケージ保存 したくない場合は、まず `.ssproj` を作ってから再度 `Ctrl+S` すると 作業保存 になります。

#### Q. `PKG` バッジが消えない

`PKG` は「作業保存 より パッケージ（`.ssproj`）が古い」状態です。`Ctrl+Shift+S` で パッケージ保存 すると消えます。

#### Q. `*` バッジが残っている

作業保存 が未保存の状態です。`Ctrl+S` で保存すれば消えます。

### 2.3 カメラ / 構図

#### Q. ショットカメラ を切替えても ビューポート の視点が変わらない

現在のモードが **ビューポートモード** になっている可能性があります。パイメニュー の **カメラ/ビューポート** で カメラモード に切替えてください。

#### Q. カメラプロパティ の Roll が編集できない

ロールロック が ON でも Roll 自体は編集可能です。ただし orbit 中は roll が抑制されます。orbit で roll を変えたいなら ロールロック を OFF にしてください。

#### Q. orthographic で ショットカメラ を保存したい

orthographic は **ビューポートのみ**です。ショットカメラ には昇格しません。orthographic で「スナップショット的な視点」を保存したい場合、ビューポート で orthographic のまま構図を作り、その状態を現行の仕様では ショットカメラ には保存できません（将来の拡張対象）。

### 2.4 用紙 / フレーム

#### Q. 紙面サイズ を変えたら フレーム の位置がずれた

アンカー と中心を固定したまま 紙面サイズ だけを変える仕組みにより、フレーム は紙面基準で自動再配置されます。見た目が「ずれた」ように見えても、紙面上の相対位置は保たれているはずです。アンカー を変えると、どの点が固定されるかが変わります。

#### Q. フレーム を増やしたら フレームマスク が 軌道 に変わった

これは意図した動作です。1 → 2 フレーム に増えた瞬間、マスク形状が `bounds` なら `trajectory` に自動昇格します（プロジェクト読み込み時を除く）。軌道 の 書き出し起点も自動で最適な角が選ばれます。

#### Q. 軌道ハンドル が ビューポート に出ない

軌道編集トグル が OFF か、軌道モード が `line` になっている可能性があります。spline モードにして、フレーム セクションの 軌道編集 を ON にしてください。

### 2.5 下絵

#### Q. 下絵 が ビューポート に表示されない

次を確認してください。

- `R` キーでプレビューが有効になっているか（ビューポート右上の HUD やツールバーの状態）
- アクティブな プリセット に アイテム が入っているか
- 該当 アイテム の **プレビュー表示** が ON か
- 該当 アイテム の画像ソースが読み込めているか（壊れたファイルパスではないか）

#### Q. 下絵 の位置が ショットカメラ ごとに違ってしまう

ショットカメラ ごとの下絵差分が効いています。ある ショットカメラ で アイテム の位置を動かすと、その ショットカメラ 専用の差分として記録されるためです。元の共有位置に戻したい場合は、下絵プロパティ で値を元に戻してください。

#### Q. `(blank)` プリセット に読み込んだのに、別の プリセット ができていた

これは意図した動作です。`(blank)` 選択中に読み込むと、新しい プリセット が自動作成され、そこに読み込まれます（`(blank)` 自体を汚さないため）。

### 2.6 ビューポート とツール

#### Q. パイメニュー が出ない

- テキスト入力にフォーカスが当たっていると パイメニュー は展開しません
- 中ボタンがないマウスの場合、タッチデバイスでは 320 ms 長押しで展開します

#### Q. 変形 Gizmo が出ない

- 変形ツール（`T`）に切替わっているか
- アセットが選択されているか
- 選択アセットが表示状態か（非表示だと Gizmo は出ない）

#### Q. 測定ツール の Enter でスケールが掛からない

次の全てが満たされる必要があります。

- 2 点とも設定されている
- アセットが選択されている
- 入力値が有限数かつ > 0

### 2.7 スプラット編集

#### Q. `Shift+E` を押しても スプラット編集 に入れない

対象となる スプラット アセットが 0 個の場合、スプラット編集 は起動できません。スプラット アセットを 1 つ以上シーンに追加してから再試行してください。

#### Q. モデル アセットを スプラット編集 できない

スプラット編集 は **スプラット アセットのみ**対応です。モデル（glTF / glb）には適用できません。

#### Q. モードを抜けたら選択が消えた

仕様です。選択状態は スプラット編集 モード中だけ保持され、モードを抜けた時点でリセットされます。編集結果（Delete / Separate / Duplicate / Transform）は永続的に反映されます。

#### Q. Separate と Duplicate の違いは？

- **Separate** — 選択 スプラット を**元アセットから抜いて**新アセットに移す（元が減る）
- **Duplicate** — 選択 スプラット を**コピーして**新アセットを作る（元は変わらない）

どちらも新アセットは元の直上に挿入されます。

### 2.8 書き出し

#### Q. Download Output ボタンが押せない

- 他の 書き出し が実行中
- `selected` 対象で 1 つも ショットカメラ が選ばれていない

後者の場合、対象を `current` / `all` に変えるか、ショットカメラ を少なくとも 1 つチェックしてください。

#### Q. PSD を開いたら マスクレイヤー が見えない

マスクレイヤー は意図的に **非表示**で出力されます。Photoshop で表示を ON にすれば見えます（不透明度 0.8）。

#### Q. PNG の色が PSD と少し違う

CAMERA_FRAMES の PNG は**線形空間**で合成されるため、通常の sRGB 合成とは色味が微妙に違う場合があります（線形の方が物理的に正確）。

#### Q. 書き出し が途中で止まる

- スプラット の描画安定化（約 1.5 秒のウォームアップ）がタイムアウトしている可能性があります。大量の スプラット アセットがある場合に発生しやすい
- メモリ不足の場合もあります（特に サイズの大きい スプラット + PSD スプラットレイヤー 化 ON）

### 2.9 履歴 / Undo

#### Q. `Ctrl+Z` で戻せない

- テキスト入力中はブラウザ側の undo が優先されます
- そもそも履歴に記録されない操作（軌道編集トグル、パイメニューの開閉など）は undo 対象外
