# 書き出し

CAMERA_FRAMES の **書き出し** は、ショットカメラ単位で紙面（用紙）に沿った画像を書き出す機能です。出力は **PNG** または **PSD**（レイヤー付き）。

インスペクターの書き出しタブで設定し、**出力** セクションの **Download Output** ボタンで実行します。

## 1. 出力セクション

![書き出し > 出力セクション](../assets/screenshots/ja/export-output-section.png)

### 1.1 書き出し対象

| 対象 | 意味 |
|---|---|
| **`current`** | 現在アクティブなショットカメラのみを書き出し（デフォルト） |
| **`all`** | ワークスペース内の全ショットカメラを書き出し |
| **`selected`** | 明示的にチェックしたショットカメラのみを書き出し |

`selected` 選択時はショットカメラのチェックリストが出現し、対象を選びます。何も選択されていない状態では `Download Output` ボタンが無効になります。

### 1.2 下絵を含める

チェックボックス。書き出し実行単位で、下絵を出力に含めるかを切り替えます。

### 1.3 書き出し実行

**Download Output** ボタン。以下の時に無効:

- 他の書き出しが実行中
- `selected` 対象で 1 つも選ばれていない

## 2. 書き出し設定セクション（ショットカメラごと）

ショットカメラごとに異なる設定を持てます。

![書き出し設定セクション](../assets/screenshots/ja/export-settings-section.png)

### 2.1 書き出し名（ファイル名テンプレート）

| 設定 | 例 |
|---|---|
| テンプレート | `cf-%cam` |
| 変数 | `%cam` → ショットカメラの名前 |

**ファイル名の正規化**:

- `\/:*?"<>|` と連続空白 → `-` に置換
- 連続ハイフン `-+` → `-` に圧縮
- 先頭末尾のハイフンを削除

**複数出力時の連番接尾辞**: 同じベース名が複数ある場合のみ `-01`、`-02` のような連番が付与されます（1 つだけなら接尾辞なし）。

### 2.2 書き出し形式

| 形式 | 拡張子 | 用途 |
|---|---|---|
| **`png`** | `.png` | 合成済み 1 枚画像 |
| **`psd`** | `.psd` | レイヤー分け画像（Photoshop 形式） |

### 2.3 グリッドオーバーレイ

チェックボックス。ON / OFF の切替。ON のときに **グリッドレイヤーモード** が現れます。

### 2.4 グリッドレイヤーモード（グリッドオーバーレイ ON 時のみ）

| モード | 効果（PSD 構造上の位置） |
|---|---|
| **`bottom`** | グリッドレイヤーが Render より**下**に配置（背景より上） |
| **`overlay`** | グリッドレイヤーが Render より**上**に配置（最上部寄り） |

PNG では単に合成順に影響します。

### 2.5 モデルレイヤー（PSD 形式のみ）

チェックボックス。

- ON — PSD にモデルアセットを個別レイヤーとして出力
- OFF — モデルも背景に統合される

### 2.6 スプラットレイヤー（PSD + モデルレイヤーの両方が ON の時のみ）

チェックボックス。

- ON — スプラットアセットも個別レイヤーとして出力
- OFF — スプラットは統合される

モデルレイヤーが OFF の場合、自動的に無効になります。

## 3. 書き出し実行フロー

### 3.1 開始

**Download Output** ボタンを押すと、UI が一時的に操作できなくなり、進捗オーバーレイが表示されて書き出しが始まります。

### 3.2 フェーズ

進捗オーバーレイには現在の処理フェーズが表示されます。

![書き出し実行中の進捗オーバーレイ](../assets/screenshots/ja/export-progress.png)

| フェーズ | 内容 |
|---|---|
| **準備** | 書き出し可能かチェック、前処理 |
| **メイン描画** | ショットカメラからのメインレンダリング |
| **ガイド** | グリッドと水平線（グリッドオーバーレイ ON 時） |
| **マスク** | フレームマスク（マスクあり時） |
| **PSD ベース合成** | PSD 書き出し時の下地合成 |
| **モデルレイヤー** | モデルアセットをレイヤー化（オプション ON 時） |
| **スプラットレイヤー** | スプラットアセットをレイヤー化（オプション ON 時） |
| **下絵** | 下絵画像のレイヤー化（下絵を含める ON 時） |
| **書き込み** | ファイルに書き出し |

### 3.3 完了

ファイルがダウンロードされ、書き出し完了のステータスメッセージが表示されます。

### 3.4 エラー

失敗時はエラー内容が表示されます。

## 4. PNG 書き出し

### 4.1 構成

PNG は全レイヤーを **合成した 1 枚画像**として書き出されます。合成順は PSD と同じルール（グリッドレイヤーモードなどを反映）。

### 4.2 合成の特徴

CAMERA_FRAMES の PNG 合成は **線形空間**で行われるため、通常の sRGB 合成で起きる境界部の色ずれが抑えられ、物理的に正しい合成結果が得られます。

## 5. PSD 書き出し

### 5.1 レイヤー構造

PSD は **下から上** のスタック順で格納されます。Photoshop で開くと**上から**表示されます。

レイヤー順序（下から上）:

1. **Background**（あれば）
2. **下絵（背面）**（グループ）— グリッドレイヤーモード = `bottom` の時
3. **Grid** — グリッドレイヤーモード = `bottom` の時
4. （グリッドレイヤーモード = `overlay` の場合は背面の下絵のみ先に出力）
5. **Render** — メインレンダリング
6. **スプラットレイヤー**（反転順）— スプラットレイヤー ON 時
7. **モデルレイヤー**（反転順）— モデルレイヤー ON 時
8. **Grid** — グリッドレイヤーモード = `overlay` の時
9. **Eye Level**
10. **下絵（前面）**（グループ）
11. **フレーム**（グループ）
    - フレームオーバーレイレイヤー
    - 軌道レイヤー（`trajectoryExportSource` ≠ `none` の時）
12. **マスク**（非表示、不透明度 0.8、フレームマスクの形状に応じた描画）

### 5.2 主要レイヤーの詳細

#### Render

ショットカメラから見たメインの描画結果（スプラット + モデルを統合した通常描画）。

#### モデルレイヤー

各モデルアセットを個別レイヤーとして出力。反転されているので、シーンマネージャーの並びが PSD の上から順に一致します。

#### スプラットレイヤー

同上のスプラットアセット版。

#### 下絵（背面 / 前面）

プリセットのアイテムが、`back` / `front` グループごとにレイヤーグループとして格納されます。各レイヤーの `left` / `top` / `opacity` が反映されます。

#### グリッド / 水平線

ビューポートでガイドとして見えていたグリッドと水平線が、別レイヤーとして出力されます。

#### フレームグループ

各フレームの枠線や注釈が `フレーム` グループに入ります。`trajectoryExportSource` が `none` 以外なら、同じグループに軌道レイヤーが追加されます。軌道レイヤーの名前は起点によって変わります（`trajectory`, `Trajectory Top Left` 等）。

#### マスク（非表示）

フレームマスクが出ていた場合、その領域を示すマスクレイヤーが**非表示かつ不透明度 0.8** で追加されます。必要なときだけ Photoshop 側で表示を ON にして使います。

### 5.3 メタデータ

PSD ファイルには解像度 150 DPI、単位インチ、サムネイル（最大 256 px）が埋め込まれます。

## 6. フレームマスクの書き出し反映

### 6.1 マスクモードと対象フレーム

| モード | 書き出し |
|---|---|
| `off` | マスクなし |
| `all` | 全フレームの外側をマスク |
| `selected` | 選択フレームのみマスク |

### 6.2 形状

| 形状 | 意味 |
|---|---|
| `bounds` | フレーム矩形のバウンディング（1 フレームならそのまま、複数なら共通の向きでまとめたバウンディング、向きがバラバラなら軸並行のバウンディング） |
| `trajectory` | フレーム軌道の掃引領域 |

### 6.3 軌道書き出し起点

| 起点 | PSD 出力 |
|---|---|
| `none` | 軌道レイヤー なし |
| `center` | `trajectory`（フレーム中心起点） |
| `top-left` | `Trajectory Top Left`（フレーム左上角起点） |
| `top-right` | `Trajectory Top Right` |
| `bottom-right` | `Trajectory Bottom Right` |
| `bottom-left` | `Trajectory Bottom Left` |

各フレームの起点に、**軌道線と直交する目盛り** が同じレイヤーに描かれます。

## 7. 書き出し前の状態要件

### 7.1 スプラットの描画安定化

スプラットアセットがある場合、書き出しの直前に表示品質を安定させるための短いウォームアップ処理（最大 1.5 秒ほど）が自動で入ります。

### 7.2 アクティブなショットカメラ

`current` 対象では、アクティブなショットカメラが無いと書き出しできません（エラー表示）。

### 7.3 下絵の可用性

下絵を含める設定が ON でも、画像ファイルが取得できないアイテムはレイヤーに出ません（該当ファイルはスキップ）。

## 8. 元に戻す / やり直し

書き出し設定の変更はすべて履歴に記録されており、`Ctrl+Z` / `Ctrl+Y` で巻き戻し / やり直しできます。
